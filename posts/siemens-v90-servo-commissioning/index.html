SINAMICS V90 是西门子面向通用运动控制的经济型伺服驱动，搭配 SIMOTICS S-1FL6 伺服电机，广泛用于定位、收放卷、贴标、包装等场合。本文按实际调试顺序，系统梳理 V90 从硬件接线 → 版本选型 → 参数设置 → 试运行 → 与 PLC 通讯组态的完整流程，覆盖脉冲（PTI）和 PROFINET（PN）两种主流版本。

一、V90 版本与选型#

V90 分为两大硬件版本，调试方式差别很大，动手前必须先确认手里是哪一种：

版本 接口 控制方式 调试软件
V90 PTI（脉冲版） 脉冲/方向输入、模拟量 外部脉冲定位、模拟量调速、内部定位块 V-ASSISTANT
V90 PN（总线版） PROFINET 经 PN 报文由 PLC 控制（位置/速度） V-ASSISTANT + TIA Portal
  • PTI 版：靠 PLC 高速脉冲输出（PTO）发脉冲控制，或用驱动内部的 IPos 定位块。接线简单，成本低，适合简单定位。
  • PN 版：通过 PROFINET 与 S7-1200/1500 通讯，配合工艺对象（TO）做精确运动控制，是目前主流。本文重点讲 PN 版与 PLC 的组态。

电机与驱动还要匹配功率、电压等级（220V/400V）、是否带抱闸、编码器类型（增量/绝对值），订货号需对应。

二、硬件接线#

2.1 主回路#

端子 说明
L1/L2/L3（或 L1/L2） 主电源输入，注意电压等级 220V 单相/三相 或 400V 三相
U/V/W 接伺服电机动力线，相序必须对应，接反会导致失控或报警
DCP/DCN 直流母线 / 外接制动电阻（大惯量、频繁启停时需外接）
PE 保护接地，务必可靠接地，否则编码器易受干扰报警

2.2 控制与编码器#

  • 编码器线：连接电机编码器到驱动 X9（注意增量式与绝对值电机线不同）。
  • 抱闸：带抱闸电机需接 24V 抱闸电源，由驱动控制时序自动释放/抱紧。
  • STO 安全端子：X8 的 STO1/STO2 出厂有短接片，不用安全功能时必须保留短接，否则驱动报 STO 激活无法使能。
  • PTI 版：脉冲/方向接 X8 的高速输入；数字量 IO 接 DI/DO。
  • PN 版：两个 PROFINET 口（X150 P1/P2），接到 PLC 或交换机。

新手高频坑：STO 短接片缺失、PE 不接地、U/V/W 与编码器不匹配（不同电机功率混接），是上电报警的三大主因。

三、V-ASSISTANT 软件调试#

V-ASSISTANT 是 V90 专用调试软件，通过 Mini-USB 或以太网 连接驱动。调试主线：

3.1 连接与选择电机#

  1. USB/网线连接驱动，打开 V-ASSISTANT，在线连接到 V90。
  2. 软件自动识别电机（带电子铭牌的电机会自动读出型号）；若未识别需手动选型号。
  3. 确认驱动与电机订货号匹配。

3.2 选择控制模式#

在「选择驱动」→ 设置控制模式，常用：

控制模式 代号 说明 适用
外部脉冲位置控制 PTI 接收外部脉冲定位 PTI 版，PLC 发脉冲
内部位置控制 IPos 驱动内部定位块（最多 8 段） PTI 版，简单多段定位
速度控制 S / Speed 模拟量或固定转速 调速场合
转矩控制 T 转矩给定 张力控制
PROFINET 控制 — 由 PLC 经报文控制 PN 版

PN 版选 PROFINET 模式后，位置/速度由 PLC 通过报文给定，V-ASSISTANT 主要用于参数设置、优化和监控。

3.3 基本参数设置#

  • 机械结构 / 电子齿轮比：设置电机转一圈对应的负载位移（丝杠导程、减速比），换算电子齿轮比，保证位置指令与实际位移一致。
  • 回零方式：选择参考点/挡块/编码器零脉冲等回零模式。
  • JOG 速度、加减速时间、限位：设置点动速度、斜坡时间、软/硬限位。
  • 数字量 IO 功能：分配 DI（使能 SON、报警复位 RESET、正反转限位 CWL/CCWL 等）和 DO（就绪、报警、定位完成等）。

3.4 JOG 试运行（关键一步）#

参数设好后，务必先在 V-ASSISTANT 里 JOG 点动试运行，确认：

  1. 电机能正常使能、转动方向正确、无异响；
  2. 编码器反馈正常、无跟随误差报警；
  3. 抱闸释放/抱紧时序正常。

JOG 通过说明接线和基本参数 OK，再进入与 PLC 的联调。

3.5 一键优化（增益整定）#

V-ASSISTANT 提供一键自动优化：根据负载惯量比自动整定位置/速度环增益。

  • 先做惯量辨识，得到负载惯量比；
  • 选择动态等级（保守/标准/动态），软件自动设增益；
  • 试运行观察是否有振动、过冲，必要时手动微调刚性等级。

四、V90 PN 与 S7-1200/1500 的 PROFINET 组态#

这是 PN 版的核心。V90 PN 作为 PROFINET IO 设备接入，由 PLC 通过**报文（Telegram）**交换控制字/状态字与设定值/实际值。

4.1 安装 GSD 文件#

TIA Portal 需先安装 V90 PN 的 GSDML 文件（选项 → 管理通用站描述文件），之后才能在硬件目录里找到 V90。

4.2 添加设备并组网#

  1. 网络视图中添加 V90 PN，拖到 PROFINET 网络上，与 PLC 的 PN 口连成同一子网。
  2. 分配 IP 地址 和 PROFINET 设备名称（设备名必须与在线分配给驱动的名称一致，否则连接不上）。
  3. 通过「在线 → 分配设备名称」把组态名称写入实际驱动。

4.3 选择报文类型#

V90 PN 支持的常用报文：

报文 用途 说明
标准报文 1 速度控制 16 位转速给定
标准报文 3 位置/速度控制 带编码器实际值，配合工艺对象做位置控制
西门子报文 111 基本定位器 EPOS 驱动内部定位器，PLC 给目标位置即可
  • 用 TIA Portal 工艺对象（TO_PositioningAxis）+ 标准报文 3：位置控制在 PLC 侧由 TO 完成，最常用、最灵活。
  • 用 报文 111 + EPOS：定位在驱动内部完成，PLC 只下目标位置和启动命令，PLC 侧编程更简单。

4.4 配置工艺对象（标准报文 3 方案）#

  1. 在 PLC 项目中添加工艺对象 → TO_PositioningAxis（定位轴）。
  2. 在 TO 的「硬件接口」里，驱动选择已组态的 V90 PN，数据连接选标准报文 3。
  3. 设置编码器、机械参数（电子齿轮、丝杠导程）、动态默认值（速度/加减速）、限位等，需与 V-ASSISTANT 中的机械设置一致。
  4. 下载后用 TO 的调试面板做轴的使能、JOG、绝对/相对定位、回零测试。

4.5 PLC 侧运动控制指令#

工艺对象配好后，用 MC（Motion Control）指令 控制轴（位于 工艺 → Motion Control）：

指令 功能
MC_Power 轴使能 / 禁用
MC_Reset 故障复位
MC_Home 回零
MC_MoveJog 点动
MC_MoveAbsolute 绝对定位
MC_MoveRelative 相对定位
MC_MoveVelocity 速度运行
MC_Halt 停止

典型使能 + 定位逻辑（SCL 伪代码）：

// 1. 上电使能
"MC_Power"(Axis := "Axis_1", Enable := axisEnable, StopMode := 0);

// 2. 故障复位（上升沿）
"MC_Reset"(Axis := "Axis_1", Execute := resetTrig);

// 3. 回零（首次运行先回零）
"MC_Home"(Axis := "Axis_1", Execute := homeTrig, Mode := 3);

// 4. 绝对定位到 100mm，速度 50mm/s
"MC_MoveAbsolute"(Axis     := "Axis_1",
                  Execute  := moveTrig,
                  Position := 100.0,
                  Velocity := 50.0);
IF "MC_MoveAbsolute".Done THEN
    // 定位完成
    ;
END_IF;

调试顺序建议：MC_Power 使能成功（Status=TRUE）→ MC_Reset 清故障 → MC_Home 回零 → 再做 MC_MoveAbsolute/Relative。先在 TO 调试面板验证，再写到程序里。

五、常见报警与排查#

V90 故障码以 F（故障） 和 A（报警） 开头，面板/软件显示。高频项：

报警/现象 含义 排查方向
无法使能、报 STO STO 安全端子未短接/未激活 检查 X8 STO1/STO2 短接片或安全回路
F7801 / 过流 U/V/W 接线、电机短路、参数错 检查动力线、电机匹配、惯量比
F52xx 编码器故障 编码器线松动、干扰、电机不匹配 检查 X9 编码器线、屏蔽接地、PE
F1911 等通讯故障 设备名/IP 不符、网线、报文不一致 核对 PROFINET 设备名、IP、报文类型
跟随误差过大 增益偏低、加减速太快、机械卡滞 重新一键优化、降低动态、查机械
振动 / 啸叫 增益过高、惯量比不准 重新惯量辨识，降低刚性等级
定位不准 / 比例不对 电子齿轮比 / 机械参数错 核对丝杠导程、减速比、TO 与驱动一致

排查口诀#

  1. 先查使能链：STO 短接、SON 使能信号、无未复位故障 —— 不使能一切免谈。
  2. 再 JOG 验证：V-ASSISTANT 里点动通过，说明接线和电机参数 OK。
  3. 后通讯组态（PN 版）：设备名/IP/报文三者与 TIA Portal 一致。
  4. 比例与精度：电子齿轮、机械参数在 V-ASSISTANT 与 TO 两侧务必一致。
  5. 动态优化：先惯量辨识再一键优化，振动就降刚性，过冲就降动态。

六、调试流程速查#

完整调试一条龙，照着走基本不会乱：

  1. 接线检查 —— 主回路、U/V/W、编码器、PE、STO 短接片。
  2. 上电 / 连接 V-ASSISTANT —— 识别电机，确认订货号匹配。
  3. 选控制模式 —— PTI/IPos/Speed 或 PROFINET。
  4. 设基本参数 —— 电子齿轮、回零、JOG 速度、IO 功能。
  5. JOG 试运行 —— 验证方向、反馈、抱闸。
  6. 一键优化 —— 惯量辨识 + 增益整定。
  7. （PN 版）PROFINET 组态 —— 装 GSD、分配设备名/IP、选报文。
  8. 工艺对象 + MC 指令 —— TO 调试面板验证，再写 PLC 程序。
  9. 联调与精度校验 —— 定位精度、重复定位、限位、急停。

结语#

V90 伺服调试的核心思路是分层验证、由内到外：先在 V-ASSISTANT 里把驱动本身（接线、电机、JOG、增益）调好，确认电机能稳定转动；再接入 PLC 做 PROFINET 通讯与运动控制。最容易卡住的几个点 —— STO 短接片、PROFINET 设备名一致、电子齿轮比两侧一致、先回零再定位 —— 记牢这几条，配合「使能链 → JOG → 通讯 → 精度 → 优化」的排查顺序，V90 的调试就会非常顺畅。PN 版搭配 TIA Portal 工艺对象和 MC 指令，能把伺服运动控制做得既精确又易维护。
