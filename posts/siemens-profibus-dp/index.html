PROFIBUS DP 是西门子推广多年、至今仍大量在役的经典现场总线。虽然 PROFINET 已成主流，但大量存量设备、变频器、分布式 IO 仍以 DP 接口为主。本文讲解西门子 PLC 的 PROFIBUS DP 通讯组态，覆盖 S7-1200（经 CM 1243-5） 与 S7-1500（集成/CM DP 口） 两种平台。

一、PROFIBUS DP 基础#

1.1 协议特点#

PROFIBUS DP（Decentralized Periphery，分布式外设）是基于 RS-485 的主从式现场总线：

  • 主从结构：一个 DP 主站（Class 1）轮询多个 DP 从站，从站被动应答。
  • 站地址：每个站 0~125，主站常用 1 或 2，地址唯一不可重复。
  • 传输介质：屏蔽双绞线（紫色 PROFIBUS 电缆），最高 12 Mbps（速率越高，距离越短）。
  • 总线两端必须接终端电阻（PROFIBUS 专用接头自带拨钮，两端站点拨到 ON）。

1.2 与 PROFINET 对比#

对比项 PROFIBUS DP PROFINET IO
物理层 RS-485 双绞线 工业以太网
寻址 站地址 0~125 设备名 + IP
速率 ≤12 Mbps 100 Mbps 起
终端电阻 必须，两端 ON 无需
诊断 有，相对有限 强，通道级
现状 存量为主 新建主流

1.3 S7 平台的 DP 支持#

平台 DP 主站实现
S7-1200 本体无 DP，需加 CM 1243-5（DP 主站模块）
S7-1500 部分 CPU 集成 DP 口；或加 CM 1542-5 通讯模块
从站方 ET 200（DP 版接口模块）、G120/V20 变频器、第三方 DP 设备等

S7-1200 还有 CM 1242-5（DP 从站模块），让 1200 作为 DP 从站接入其他主站；本文以 1200 作主站（CM 1243-5）为主。

二、硬件与接线#

2.1 模块安装#

  • S7-1200：CM 1243-5 安装在 CPU 左侧（通讯模块都在 CPU 左边），最多 3 个。
  • S7-1500：集成 DP 口直接用；或在机架加 CM 1542-5。

2.2 PROFIBUS 电缆与接头#

  • 使用紫色 PROFIBUS 专用电缆（A 型，屏蔽双绞）。
  • 使用 PROFIBUS DB9 总线连接器（带终端电阻拨钮）。
  • 总线两端的站：连接器终端电阻拨到 ON；
  • 中间站：拨到 OFF（电阻断开，信号继续传递）。
  • 屏蔽层要可靠接地。

新手高频坑：终端电阻没拨对（两端没 ON、或中间误拨 ON）是 DP 通讯不稳定/不通的头号原因。

三、TIA Portal 组态步骤#

3.1 安装从站 GSD（如需）#

西门子 ET 200、G120 等常见从站 TIA Portal 已自带。第三方 DP 设备需安装 GSD 文件（选项 → 管理通用站描述文件，PROFIBUS 是 .gsd/.gse 等扩展名）。

3.2 添加主站模块并建 DP 网络#

  1. S7-1200 设备视图中，在 CPU 左侧插入 CM 1243-5。
  2. 进入网络视图，会看到 CM 1243-5 的 DP 接口（紫色）。
  3. 从硬件目录拖出 DP 从站（如 ET 200 DP 版、G120 的 DP 控制单元）。
  4. 用鼠标从 CM 1243-5 的 DP 口 拖线到从站的 DP 口，建立 DP Mastersystem（紫色总线）。

3.3 设置站地址与波特率#

  1. 选中 CM 1243-5 的 DP 接口 → 属性 → PROFIBUS 地址：设主站地址（如 2）。
  2. 每个从站设唯一站地址（如 3、4、5…），必须与从站硬件上的拨码/参数一致。
  3. 在 PROFIBUS 子网属性里设 传输速率（波特率），全网必须一致（如 1.5 Mbps）。

从站地址：ET 200 等通过硬件拨码开关或软件设定；变频器（G120）在参数里设 DP 地址。组态地址必须与实物一致。

3.4 配置从站 IO / 报文#

  • ET 200 DP：进设备视图，按实物插入 DI/DO/AI/AO 模块。
  • 变频器 G120：选择报文类型（如标准报文 1/352），用于交换控制字/状态字与设定值/实际值。

3.5 IO 地址映射#

和 PROFINET 一样，组态后系统自动给从站模块分配 I/Q 地址，程序里直接用这些地址访问，与本体 IO 用法相同：

// 读 DP 从站上的现场输入
IF "I20.0" THEN        // ET 200 DP 站的 DI
    "Q20.0" := TRUE;   // ET 200 DP 站的 DO
END_IF;

3.6 编译下载#

编译下载到 PLC，主站开始轮询各 DP 从站，进入循环数据交换。

四、实战示例：S7-1200 + CM 1243-5 + 一台 ET 200 DP#

  1. CPU 左侧插 CM 1243-5，主站地址设 2。
  2. 网络视图拖出 ET 200 DP 接口模块，DP 地址设 3（与硬件拨码一致），连到 CM 1243-5 的 DP 口。
  3. ET 200 设备视图插入 16DI / 16DO 模块。
  4. 子网波特率设 1.5 Mbps；总线两端接头终端电阻拨 ON。
  5. 记下自动分配地址：DI→I20.0~I21.7，DO→Q20.0~Q21.7。
  6. 编译下载，程序里直接用 I/Q 地址。

五、常见问题与诊断#

现象 可能原因 排查方向
从站不上线 / SF/BF 灯亮 站地址不符、终端电阻、断线 核对地址、查两端电阻 ON、查电缆接头
通讯时通时断 终端电阻错、干扰、波特率过高 修正电阻、屏蔽接地、降波特率
组态与实际不符 模块/报文型号不一致 对照实物核对模块订货号、报文
全网都不通 主站地址冲突、波特率不一致 全网波特率一致、地址唯一
找不到从站 GSD 未装、从站未供电 装 GSD、查从站电源

诊断手段#

  • CM/CPU 的 SF（系统故障）、BF（总线故障）LED：BF 闪烁通常指总线/从站问题。
  • 在线 & 诊断：查看 DP 主站系统状态、各从站在线情况。
  • 诊断中断 OB：组态后从站故障可触发 OB82（诊断中断）、从站掉站触发 OB86（机架/站故障）。

排查口诀#

  1. 先查电阻：总线两端接头终端电阻 ON，中间 OFF。
  2. 再核地址：每站地址唯一，组态与硬件拨码/参数一致。
  3. 统一波特率：全网波特率相同。
  4. 对组态：从站模块/报文与实物对应。
  5. 看灯 + 诊断 OB：BF/SF 灯 + OB82/OB86 定位故障站。

六、要点速查#

环节 关键点
主站实现 S7-1200 加 CM 1243-5；S7-1500 集成口/CM 1542-5
介质 紫色 PROFIBUS 电缆 + DB9 终端电阻接头
终端电阻 两端站 ON，中间站 OFF（头号坑）
地址 站地址 0~125 唯一，组态=硬件
波特率 全网一致
IO 使用 自动分配 I/Q 地址，像本体 IO 一样用
诊断 BF/SF 灯 + OB82/OB86

结语#

PROFIBUS DP 虽是「老」总线，但在工厂存量设备和变频器场景里仍随处可见。S7-1200 靠 CM 1243-5 即可作 DP 主站，组态思路与 PROFINET 高度一致 —— 建网络、设地址、配模块、用 I/Q。与 PROFINET 最大的不同在物理层：站地址要唯一、波特率要全网一致、终端电阻两端必须 ON，记牢这三条，DP 通讯的绝大多数问题都能避开。新建项目优先选 PROFINET，但维护老设备时，吃透 DP 仍是必备技能。
