Modbus TCP 是 Modbus 协议在以太网上的实现，把原本运行于 RS-485/RS-232 的 Modbus 报文封装进 TCP/IP，借助以太网实现更高速率、更长距离和更灵活的组网。本文聚焦 S7-1200 / S7-1500（TIA Portal） 平台，系统讲解 Modbus TCP 的指令用法、连接组态与实战要点。S7-200 SMART 不在本文范围内。

一、Modbus TCP 基础#

1.1 与 Modbus RTU 的区别#

Modbus TCP 与 RTU 的数据模型（线圈、寄存器、功能码）完全一致，差别在于传输层封装：

对比项 Modbus RTU Modbus TCP
物理/链路层 RS-485 / RS-232 串行 以太网 TCP/IP
寻址方式 从站地址 1~247 IP 地址 + 端口 502
校验 CRC 16 由 TCP 保证，无 CRC
报文头 无 多 7 字节 MBAP 头
速率 通常 ≤115200 bps 10/100 Mbps 起
拓扑 单主多从、半双工总线 交换式以太网、多客户端并发

1.2 MBAP 报文头#

Modbus TCP 用 7 字节的 MBAP（Modbus Application Protocol）头 取代了 RTU 的 CRC：

字段 字节数 说明
事务标识 Transaction ID 2 请求/应答配对，由客户端递增
协议标识 Protocol ID 2 Modbus 固定为 0
长度 Length 2 后续字节数
单元标识 Unit ID 1 类似从站地址，网关后挂串行设备时用，直连设备常为 1 或 255

后面紧跟功能码与数据，与 RTU 相同。

1.3 功能码与地址模型#

功能码、四类数据区、以及「40001 偏移」陷阱与 Modbus RTU 完全一致，这里仅快速回顾：

数据区 逻辑地址 功能码
线圈 Coils 00001 ~ 01 / 05 / 15
离散输入 Discrete Inputs 10001 ~ 02
输入寄存器 Input Registers 30001 ~ 04
保持寄存器 Holding Registers 40001 ~ 03 / 06 / 16

协议地址从 0 开始，手册的「40001」对应协议地址 0。S7-1200/1500 的 Modbus TCP 指令中，MB_DATA_ADDR 同样直接填 40001 这类逻辑地址。

二、S7-1200/1500 的两条核心指令#

Modbus TCP 不需要额外的通讯模块，直接用 CPU 本体或 CP 模块的**以太网口（PROFINET 口）**即可。指令位于指令树 通信 → 其他 → MODBUS TCP：

  • MB_CLIENT：作为 Modbus TCP 客户端（主站），主动连接服务器并读写数据。
  • MB_SERVER：作为 Modbus TCP 服务器（从站），被动响应外部客户端的请求。

两条指令都需要一个 TCON_IP_v4 类型的连接参数结构来描述连接。

2.1 连接参数 TCON_IP_v4#

在 DB 中建一个 TCON_IP_v4 类型的变量，关键字段：

字段 说明
InterfaceId 本地以太网接口的硬件标识符（从系统常量选，如 Local~PROFINET_Interface_1）
ID 连接 ID，每个连接唯一（1~4095），不能与其他通讯连接冲突
ConnectionType 连接类型，Modbus TCP 固定 16#0B（11）
ActiveEstablished 客户端置 TRUE（主动建立），服务器置 FALSE（被动监听）
RemoteAddress.ADDR 远程设备 IP，4 字节数组，如 192.168.0.10
RemotePort 远程端口，Modbus 标准为 502（客户端用）
LocalPort 本地端口，服务器用 502，客户端填 0（自动分配）

2.2 MB_CLIENT —— 作为客户端（主站）#

                MB_CLIENT
            ┌──────────────────────┐
   REQ ────►│ REQ              DONE ├──► 完成
DISCONNECT─►│ DISCONNECT       BUSY ├──► 忙
    0 ─────►│ MB_MODE         ERROR├──► 错误
40001 ─────►│ MB_DATA_ADDR   STATUS├──► 状态字
   10 ─────►│ MB_DATA_LEN          │
 P#DB ─────►│ MB_DATA_PTR          │
 &CONN ────►│ CONNECT              │
            └──────────────────────┘
参数 说明
REQ 上升沿触发一次请求
DISCONNECT 0=建立并保持连接，1=断开连接
MB_MODE 模式：0=读，1=写，2=诊断（与地址一起决定功能码）
MB_DATA_ADDR Modbus 起始地址，填 40001 / 30001 / 00001 / 10001 等逻辑地址
MB_DATA_LEN 读写数据长度（位或字）
MB_DATA_PTR PLC 侧数据存放区指针（指向 DB 或 M 区）
CONNECT 指向 TCON_IP_v4 连接参数（用 &"DB名".连接变量）
DONE / BUSY / ERROR / STATUS 完成 / 忙 / 错误 / 状态字

MB_MODE 与 MB_DATA_ADDR 决定功能码：

功能码 MB_MODE MB_DATA_ADDR
01 读线圈 0 1 ~ 9999
02 读离散输入 0 10001 ~ 19999
03 读保持寄存器 0 40001 ~ 49999
04 读输入寄存器 0 30001 ~ 39999
05/15 写线圈 1 1 ~ 9999
06/16 写保持寄存器 1 40001 ~ 49999

同一个 MB_CLIENT 背景 DB 对应一条连接。DISCONNECT 保持 0 时连接会一直保持，适合频繁轮询；只有需要主动释放连接时才置 1。

2.3 MB_SERVER —— 作为服务器（从站）#

当 S7-1200/1500 需要被上位机、SCADA 或第三方主站轮询时使用：

                MB_SERVER
            ┌──────────────────────┐
DISCONNECT─►│ DISCONNECT        NDR ├──► 收到新数据（被写入）
 &HOLD ────►│ MB_HOLD_REG        DR ├──► 数据被读取
 &CONN ────►│ CONNECT         ERROR├──► 错误
            │                 STATUS├──► 状态字
            └──────────────────────┘
参数 说明
DISCONNECT 0=被动监听并接受连接，1=断开
MB_HOLD_REG 作为「保持寄存器区」的 DB 或 M 区，外部客户端读写的就是这块内存，映射为 40001 起始
CONNECT 指向 TCON_IP_v4，ActiveEstablished := FALSE、LocalPort := 502
NDR New Data Ready，客户端写入了新数据时为 1
DR Data Read，数据被客户端读取时为 1

要点：

  1. MB_HOLD_REG 指向的 DB 要设为非优化访问（取消勾选「优化的块访问」）。
  2. 服务器端 CONNECT 的 ActiveEstablished 必须为 FALSE，LocalPort 设为 502。
  3. 一个 MB_SERVER 实例服务一个连接；要支持多个客户端并发，需用多个背景 DB + 多个不同 ID 的连接。

2.4 多请求 / 多服务器轮询#

和 RTU 一样，MB_CLIENT 同一连接同一时刻只能处理一条请求，要轮询多种功能码或多台服务器：

  • 轮询同一台服务器的不同寄存器：复用同一背景 DB 与连接，用 DONE/BUSY 做状态机，切换 MB_MODE/MB_DATA_ADDR/MB_DATA_PTR 后再发 REQ。
  • 轮询多台服务器：每台一个 MB_CLIENT 背景 DB + 独立连接 ID + 独立 TCON_IP_v4（不同 RemoteAddress）。

SCL 伪代码（单连接、两条请求轮询）：

CASE step OF
    0:  // 读保持寄存器
        "MB_CLIENT_DB"(REQ := trig, DISCONNECT := 0, MB_MODE := 0,
                       MB_DATA_ADDR := 40001, MB_DATA_LEN := 10,
                       MB_DATA_PTR := "rdData", CONNECT := "conn".srv1);
        IF "MB_CLIENT_DB".DONE THEN step := 10; END_IF;

    10: // 写保持寄存器
        "MB_CLIENT_DB"(REQ := trig, DISCONNECT := 0, MB_MODE := 1,
                       MB_DATA_ADDR := 40020, MB_DATA_LEN := 5,
                       MB_DATA_PTR := "wrData", CONNECT := "conn".srv1);
        IF "MB_CLIENT_DB".DONE THEN step := 0; END_IF;
END_CASE;

三、实战示例：S7-1200 读取远程 IO 模块#

假设有一台 Modbus TCP 远程 IO 模块，IP 192.168.0.10，端口 502，其 16 路模拟量输入位于保持寄存器 40001~40016。

  1. 建连接参数：在 DB（如 ConnData）中建 TCON_IP_v4 变量 srv1：
    • InterfaceId 选本地 PN 口；ID := 1；ConnectionType := 16#0B；
    • ActiveEstablished := TRUE；RemoteAddress.ADDR := [192,168,0,10]；RemotePort := 502。
  2. 建数据区：DB 中建 aiData : Array[0..15] of Int。
  3. OB1 调用：
// 每 100ms 产生读取脉冲 trigRead
"MB_CLIENT_DB"(REQ          := trigRead,
               DISCONNECT   := 0,
               MB_MODE      := 0,            // 读
               MB_DATA_ADDR := 40001,        // 保持寄存器起始
               MB_DATA_LEN  := 16,           // 16 个字
               MB_DATA_PTR  := "DB_Data".aiData,
               CONNECT      := "ConnData".srv1);

IF "MB_CLIENT_DB".DONE THEN
    // aiData[] 中即为 16 路最新数据
    ;
END_IF;
  1. 写输出同理：把 MB_MODE := 1、MB_DATA_ADDR 指向目标保持寄存器、MB_DATA_PTR 指向写缓冲，读完成后再触发写。

四、常见错误码与排查#

STATUS 输出（16 进制）的高频值与排查方向：

STATUS / 现象 含义 排查方向
0x7000 无作业执行（空闲） 正常，REQ 未触发
0x7001/0x7002 正在建立连接 / 正在处理 正常过渡态，等待 DONE
0x8187 CONNECT 参数无效 检查 TCON_IP_v4 各字段、连接 ID 是否冲突
0x80C4 连接被对方拒绝/中断 确认对方设备开启 502 端口、网络可达
0x80B3/0x8382 端口/连接资源冲突 连接 ID、LocalPort 是否与其他连接重复
连接超时无应答 IP/端口错、网络不通、防火墙 ping 测通、核对 IP 与端口 502
非法数据地址 (0x02) 地址超范围或区号错 核对寄存器表，注意 40001 偏移
数据高低字颠倒 字节序/字序差异 32 位数据做字交换

排查口诀#

  1. 先通网络：用 ping 确认 IP 可达，确认对方 502 端口开放、无防火墙拦截。
  2. 再查连接参数：InterfaceId、连接 ID 唯一、ConnectionType=16#0B、客户端 ActiveEstablished=TRUE、RemotePort=502。
  3. 后核地址映射：功能码、寄存器地址（40001 偏移）、读写长度、Unit ID（经网关时）。
  4. 最后看时序：REQ 边沿触发，多请求用 DONE/BUSY 串行轮询；DISCONNECT 保持 0 维持长连接。

五、Modbus TCP 与 RTU 指令对照#

功能 Modbus RTU Modbus TCP
串口/连接初始化 Modbus_Comm_Load 无需，连接参数由 TCON_IP_v4 描述
主站/客户端 Modbus_Master MB_CLIENT
从站/服务器 Modbus_Slave MB_SERVER
寻址 从站地址 MB_ADDR IP + 端口 + Unit ID
触发 REQ 上升沿 REQ 上升沿
读写方向 MODE (0/1) MB_MODE (0/1/2)
地址参数 DATA_ADDR 填 40001 等 MB_DATA_ADDR 填 40001 等
数据指针 DATA_PTR MB_DATA_PTR

结语#

S7-1200/1500 的 Modbus TCP 实现比 RTU 更省事：不需要通讯模块和串口初始化，只要配好一个 TCON_IP_v4 连接参数，再用 MB_CLIENT（客户端）或 MB_SERVER（服务器）收发即可。难点主要集中在连接参数的正确填写（连接 ID 唯一、客户端/服务器的 ActiveEstablished 与端口区别）与一如既往的寄存器地址映射。遇到不通时，按「网络 ping 通 → 连接参数 → 地址映射 → 时序」的顺序排查，问题大多能很快定位。
