Modbus RTU 是工业现场最常见的串行通讯协议之一，凭借协议开放、实现简单、设备支持广泛的特点，被大量用于 PLC 与变频器、仪表、温控器、电表等设备之间的数据交换。本文系统梳理西门子两大主流平台 —— S7-1200/1500（TIA Portal） 与 S7-200 SMART —— 中 Modbus RTU 通讯指令的用法、参数含义和实战要点。

一、Modbus RTU 基础回顾#

在动手写程序之前，先理清几个绕不开的概念。

1.1 物理层与帧结构#

Modbus RTU 工作在 RS-485（或 RS-232） 串行总线上，采用主从（Master/Slave）半双工通讯：

  • 总线上只能有 一个主站，可挂接多个从站（理论 247 个，实际受驱动能力限制通常 ≤32 个）。
  • 每个从站有唯一的 从站地址（1~247），地址 0 为广播。
  • 主站发起请求，从站被动应答；从站之间不能直接通讯。

一帧 RTU 报文结构如下：

从站地址 功能码 数据区 CRC 校验
1 字节 1 字节 N 字节 2 字节

帧与帧之间以 ≥3.5 个字符的静默时间 分隔。

1.2 常用功能码#

功能码 名称 作用 操作对象
01 (0x01) 读线圈 读取多个开关量输出 线圈（可读写位）
02 (0x02) 读离散量输入 读取多个开关量输入 输入位（只读）
03 (0x03) 读保持寄存器 读取多个字寄存器 保持寄存器（可读写字）
04 (0x04) 读输入寄存器 读取多个字寄存器 输入寄存器（只读字）
05 (0x05) 写单个线圈 写一个开关量输出 线圈
06 (0x06) 写单个保持寄存器 写一个字 保持寄存器
15 (0x0F) 写多个线圈 批量写开关量 线圈
16 (0x10) 写多个保持寄存器 批量写字 保持寄存器

1.3 Modbus 地址模型与「偏移」陷阱#

Modbus 标准定义了四类数据区，协议地址从 0 开始，但设备手册常用「逻辑地址 / PLC 地址」从 1 开始，这是新手最容易踩的坑：

数据区 逻辑地址范围 协议地址 功能码
线圈 Coils 00001 ~ 09999 0x0000 起 01 / 05 / 15
离散输入 Discrete Inputs 10001 ~ 19999 0x0000 起 02
输入寄存器 Input Registers 30001 ~ 39999 0x0000 起 04
保持寄存器 Holding Registers 40001 ~ 49999 0x0000 起 03 / 06 / 16

关键点：例如手册写的「保持寄存器 40001」对应协议地址 0，「40100」对应协议地址 99。西门子指令中填的是**协议地址（从 0 开始）**还是带区号的地址，不同平台不一样，下文会逐一说明。

二、S7-1200 / S7-1500（TIA Portal）#

S7-1200/1500 通过 CM 1241（RS-485/RS-232）通讯模块 或 CB 1241（RS-485 通讯板） 实现 Modbus RTU。核心是三条指令，位于指令树 通信 → 通信处理器 → MODBUS (RTU)：

  • Modbus_Comm_Load：组态串口参数，每个端口上电时调用一次。
  • Modbus_Master：作为 Modbus 主站收发数据。
  • Modbus_Slave：作为 Modbus 从站响应外部主站。

2.1 硬件与组态步骤#

  1. 在设备组态中添加 CM 1241 (RS422/485) 或 CB 1241 (RS485) 模块。
  2. 接线：RS-485 使用 T/RA (−) 与 T/RB (+) 两根信号线，长距离需在总线两端接 120Ω 终端电阻，并注意共地。
  3. 程序里调用三条指令，背景 DB 由 TIA Portal 自动生成。

2.2 Modbus_Comm_Load —— 串口初始化#

通常放在 OB100（启动 OB）中调用一次，或用首次扫描位 FirstScan 触发。

              Modbus_Comm_Load
            ┌────────────────────┐
    REQ ───►│ REQ           DONE ├──► 完成标志
    PORT ──►│ PORT          ERROR├──► 错误标志
   9600 ───►│ BAUD         STATUS├──► 状态字
      0 ───►│ PARITY             │
      0 ───►│ FLOW_CTRL          │
   1000 ───►│ RESP_TO            │
   &DB ────►│ MB_DB              │
            └────────────────────┘

主要参数：

参数 说明 典型值
REQ 上升沿触发执行 首次扫描脉冲
PORT 通讯端口硬件标识符（从「系统常量」选择，如 Local~CM_1241...） —
BAUD 波特率 9600 / 19200
PARITY 校验：0 无，1 奇，2 偶 通常 0 或 2
FLOW_CTRL 流控，默认 0 0
RESP_TO 响应超时 (ms) 1000
MB_DB 关联到 Modbus_Master/Modbus_Slave 的背景 DB（用 &"DB名"） —

MB_DB 必须指向后续主/从指令的背景数据块，把串口和收发指令绑定在一起，这一步漏了会导致通讯不通。

2.3 Modbus_Master —— 作为主站#

              Modbus_Master
            ┌────────────────────┐
    REQ ───►│ REQ           DONE ├──► 完成
      1 ───►│ MB_ADDR       BUSY ├──► 忙
      3 ───►│ MODE         ERROR ├──► 错误
      0 ───►│ DATA_ADDR    STATUS├──► 状态字
    100 ───►│ DATA_LEN           │
   P#DB ───►│ DATA_PTR           │
            └────────────────────┘
参数 说明
REQ 上升沿发起一次请求（注意是边沿触发，不能常 1）
MB_ADDR 目标从站地址 1~247
MODE 读写方向：0=读，1=写（与 DATA_ADDR 一起决定功能码）
DATA_ADDR Modbus 起始地址，直接填带区号的逻辑地址，如 40001、30001、00001、10001
DATA_LEN 读写的位数 / 字数
DATA_PTR 数据在 PLC 中的存放指针，指向 DB 或 M 区，如 P#DB10.DBX0.0 或用符号

MODE / DATA_ADDR 与功能码的对应关系（西门子用这两者自动推导功能码）：

功能码 MODE DATA_ADDR 范围 数据长度
01 读线圈 0 1 ~ 9999 1~2000 位
02 读离散输入 0 10001 ~ 19999 1~2000 位
03 读保持寄存器 0 40001 ~ 49999 1~125 字
04 读输入寄存器 0 30001 ~ 39999 1~125 字
05/15 写线圈 1 1 ~ 9999 1 或多个位
06/16 写保持寄存器 1 40001 ~ 49999 1 或多个字

在 S7-1200/1500 中 DATA_ADDR 用的是 40001 这种带区号的逻辑地址，指令内部会自动换算成协议地址并选功能码。这点和下面 S7-200 SMART 的 Addr 参数一致。

2.4 Modbus_Slave —— 作为从站#

当 S7-1200/1500 需要被第三方主站（如触摸屏、上位机、另一台 PLC）轮询时使用：

参数 说明
MB_ADDR 本机从站地址
MB_HOLD_REG 指向作为「保持寄存器区」的 DB，外部主站读写的就是这块内存
NDR 新数据到达（主站写入了数据）
DR 数据被读取
ERROR / STATUS 错误与状态

将 MB_HOLD_REG 指向一个非优化访问的全局 DB（需在 DB 属性里取消勾选「优化的块访问」），即可把该 DB 映射为对外的 40001 起始保持寄存器区。

2.5 多请求轮询要点#

Modbus_Master 同一时刻只能处理一条请求。要轮询多个从站或多种功能码，必须：

  1. 用 BUSY / DONE 做状态机，一条完成后再发下一条；
  2. 切换 MB_ADDR、MODE、DATA_ADDR、DATA_PTR 后再给 REQ 一个上升沿；
  3. 所有读写指令共用同一个背景 DB（即都绑定到同一个 Modbus_Comm_Load）。

一个简化的轮询思路（SCL 伪代码）：

CASE step OF
    0:  // 读 1 号站保持寄存器
        "Modbus_Master"(REQ := trig, MB_ADDR := 1, MODE := 0,
                        DATA_ADDR := 40001, DATA_LEN := 10,
                        DATA_PTR := "rdData");
        IF "Modbus_Master".DONE THEN step := 10; END_IF;

    10: // 写 2 号站保持寄存器
        "Modbus_Master"(REQ := trig, MB_ADDR := 2, MODE := 1,
                        DATA_ADDR := 40001, DATA_LEN := 5,
                        DATA_PTR := "wrData");
        IF "Modbus_Master".DONE THEN step := 0; END_IF;
END_CASE;

三、S7-200 SMART#

S7-200 SMART 通过 CPU 本体的 RS-485 端口（Port 0） 或 SB CM01 信号板（Port 1） 通讯。指令通过 库（Library） 提供，位于 指令 → 库 → Modbus RTU Master / Slave，调用前需先在项目里编译一次以分配库存储区。

3.1 主站指令#

主站需要两条指令配合：

MBUS_CTRL —— 初始化（每个扫描周期调用）#

            MBUS_CTRL
          ┌──────────────┐
   EN ───►│ EN           │
   ON ───►│ Mode    Done ├──► 初始化完成
 9600 ───►│ Baud   Error ├──► 错误码
    2 ───►│ Parity       │
 1000 ───►│ Timeout      │
          └──────────────┘
参数 说明
Mode 1=启用 Modbus 协议占用端口，0=恢复为 PPI（让端口给编程口）
Baud 波特率
Parity 0 无，1 奇，2 偶
Timeout 等待从站应答的超时 (ms)
Done 初始化完成置 1
Error 错误码

MBUS_CTRL 必须每个扫描周期都执行（EN 端常通），不是只调一次。

MBUS_MSG —— 收发报文#

            MBUS_MSG
          ┌──────────────┐
  EN ────►│ EN           │
 First ──►│ First   Done ├──► 完成
    1 ───►│ Slave  Error ├──► 错误码
    3 ───►│ RW           │
40001 ───►│ Addr         │
   10 ───►│ Count        │
&VB100───►│ DataPtr      │
          └──────────────┘
参数 说明
First 上升沿触发一次发送，关键！必须用边沿信号，不能常 1，否则会反复重发
Slave 从站地址
RW 0=读，1=写
Addr Modbus 地址，填带区号逻辑地址：00001线圈 / 10001离散输入 / 30001输入寄存器 / 40001保持寄存器
Count 读写数量（位或字）
DataPtr 数据缓冲区指针，如 &VB100
Done 完成置 1
Error 错误码

多条 MBUS_MSG 轮询：同样只能逐条执行。常见做法是把多个 MBUS_MSG 串成轮询，用各自的 Done 触发下一条的 First，或用计数器/状态字循环切换。

3.2 从站指令#

MBUS_INIT —— 从站初始化（仅首次扫描调用一次）#

参数 说明
Mode 1=Modbus 从站，0=PPI
Addr 本机从站地址 1~247
Baud / Parity 波特率 / 校验
Delay 字符间附加延时
MaxIQ 可访问的 I/O 点数（0~128），一般 128
MaxAI 可访问的模拟量输入字数
MaxHold 可被主站访问的 V 区保持寄存器字数
HoldStart 保持寄存器起始地址，如 &VB0，对应主站读写的 40001

与 MBUS_CTRL 不同，MBUS_INIT 只在 首次扫描（SM0.1） 调用一次。

MBUS_SLAVE —— 从站响应（每周期调用）#

每个扫描周期执行，自动响应主站请求，输出 Done（有读写动作）与 Error。

3.3 S7-200 SMART 从站地址映射#

从站模式下，外部主站访问的地址映射到 SMART 的内存：

Modbus 地址 对应 SMART 区域
00001 起（线圈） Q 区输出
10001 起（离散输入） I 区输入
30001 起（输入寄存器） AIW 模拟量输入
40001 起（保持寄存器） 由 HoldStart 指定的 V 区

四、实战示例：S7-1200 读取变频器频率#

以读取某变频器（从站地址 1）的实际输出频率为例，假设手册标明：实际频率位于保持寄存器 40020，单位 0.01Hz。

  1. OB100 启动块：调用 Modbus_Comm_Load，PORT 选 CM1241，BAUD=9600，PARITY=2（偶校验，按变频器手册），MB_DB := &"Modbus_Master_DB"。
  2. OB1 主程序：
// 每 200ms 产生一个读取脉冲 trigRead
"Modbus_Master"(REQ      := trigRead,
                MB_ADDR  := 1,
                MODE     := 0,          // 读
                DATA_ADDR:= 40020,      // 保持寄存器 40020
                DATA_LEN := 1,          // 读 1 个字
                DATA_PTR := "freqRaw"); // INT 变量

IF "Modbus_Master".DONE THEN
    realFreq := INT_TO_REAL(freqRaw) * 0.01;  // 换算成 Hz
END_IF;
  1. 若需要写频率给定（写到 40021），把 MODE 改为 1、DATA_ADDR := 40021、DATA_PTR 指向给定值变量，并在读完成后再触发写。

五、常见错误码与排查#

各平台 STATUS / Error 输出的高频错误及含义：

错误现象 可能原因 排查方向
始终超时无应答 接线 A/B 反接、波特率/校验/从站地址不匹配 用万用表/抓包确认 A(+)、B(−)，核对参数三件套
CRC 校验错误 干扰、终端电阻缺失、波特率过高 加 120Ω 终端电阻、降波特率、信号线用屏蔽双绞
非法数据地址 (0x02) DATA_ADDR/Addr 超出从站实际地址范围或区号错 核对手册寄存器表，注意 40001 偏移
非法功能 (0x01) 从站不支持该功能码 改用从站支持的功能码（如 03 与 04 别搞混）
间歇性丢包 多主站冲突、轮询过快、未等 Done 保证单主站、用状态机串行轮询
数据高低字节颠倒 字节序 / 字序（大小端）差异 对 32 位数据做字交换，或在指令/变量侧调整

排查口诀#

  1. 先通物理层：A/B 接线、共地、终端电阻、屏蔽接地。
  2. 再对通讯参数：波特率、数据位、校验位、停止位、从站地址，主从必须完全一致。
  3. 后核地址映射：功能码、寄存器地址（注意 0 偏移）、读写长度。
  4. 最后看时序：REQ/First 必须边沿触发；多请求用 Done/BUSY 串行轮询。

六、两大平台指令对照表#

功能 S7-1200/1500 S7-200 SMART
串口/协议初始化 Modbus_Comm_Load 主站 MBUS_CTRL / 从站 MBUS_INIT
主站收发 Modbus_Master MBUS_MSG
从站响应 Modbus_Slave MBUS_SLAVE
触发方式 REQ 上升沿 First 上升沿
地址填法 DATA_ADDR 填 40001 等逻辑地址 Addr 填 40001 等逻辑地址
数据指针 DATA_PTR（P# 指针 / 符号） DataPtr（如 &VB100）
初始化调用时机 Comm_Load 仅启动时一次 CTRL 每周期 / INIT 仅首扫一次

结语#

Modbus RTU 在西门子平台上的实现并不复杂，难点几乎都集中在三处：接线与终端电阻、通讯参数一致性、寄存器地址映射（尤其是 40001 的偏移）。掌握 S7-1200/1500 的 Comm_Load + Master/Slave 三件套，以及 S7-200 SMART 的 CTRL/INIT + MSG/SLAVE 组合，再配合状态机做好多请求轮询，就能稳定地把现场仪表、变频器的数据接入 PLC。遇到不通时，按「物理层 → 通讯参数 → 地址映射 → 时序」的顺序排查，事半功倍。
